# Retail Sales Analysis Dashboard (Excel)

## 📊 Project Overview
This project transforms raw retail transaction data into an interactive Business Intelligence tool. Using a dataset of **1,000 orders**, I developed an end-to-end analysis pipeline to monitor **₹35.2L+** in sales performance, identifying seasonal trends, peak purchasing hours, and operational bottlenecks.

## 🎯 Project Goal
To convert unstructured data into a dynamic dashboard that provides stakeholders with real-time insights into high-value customer segments and seasonal revenue drivers.

## 🛠️ Technical Toolkit
* **Data Cleaning:** Utilized **Power Query** for ETL processes (handling nulls, standardizing date formats, and data normalization).
* **Data Modeling:** Established a relational star-schema between Customer, Product, and Sales tables using **Power Pivot**.
* **Advanced Analytics (DAX):** Created complex calculated measures for:
    * **Total Revenue:** ₹35,20,984
    * **Average Customer Spend:** ₹3,520.98
    * **Average Delivery Lead Time:** 5.53 Days
* **Interactivity:** Integrated **Slicers and Timelines** for granular filtering by Occasion (Diwali, Valentine's Day, etc.) and Product Category.

## 💡 Key Business Insights
* **Peak Performance:** Revenue distribution analysis identifies specific peak hours, providing a roadmap for targeted digital marketing and ad-spend optimization.
* **Seasonal Revenue Drivers:** High-impact occasions like **Anniversaries** and **Valentine’s Day** drive significantly higher average order values compared to standard days.
* **Operational Efficiency:** The average delivery time of **5.53 days** was flagged as a critical KPI, suggesting room for logistics optimization to improve customer satisfaction.
